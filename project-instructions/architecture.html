Layer 1 核心特征

📖 静态知识

Agents和Skills都是静态配置文件，Claude通过Read工具"读取"获得身份认知和能力认知。

🎭 角色定位

Agents提供"我是谁，我做什么"的角色定位和专业领域指导，不直接执行操作。

🔧 能力说明

Skills提供"能力说明，(可选)如何执行"，包含SKILL.md文档和scripts/执行引擎。

📂 文件位置

.claude/agents/ (智能体配置) 和 .claude/skills/ (技能包)，通过工具调用加载。

设计原则

指导系统架构设计的核心理念

🎯

职责分离

知识层提供"知道什么"，决策层负责"如何做"，执行层实现"真实执行"，三层职责清晰不混淆。

⚡

动态编排

Claude运行时动态发现资源、组合能力，而非依赖静态配置文件，实现灵活的任务执行。

📦

自包含设计

Skills采用自包含设计，SKILL.md + scripts/执行引擎打包在一起，便于版本控制和团队共享。

📖

渐进披露

Skills按需加载：Level 1 Metadata → Level 2 Core → Level 3 Extended → Level 4 Code，优化token使用。

🔌

可扩展性

通过MCP协议扩展外部系统集成，通过新增Agents/Skills扩展业务能力，架构开放易扩展。

🔄

持续反馈

执行结果实时反馈给Claude，支持错误处理、策略调整和迭代优化，形成闭环学习系统。
