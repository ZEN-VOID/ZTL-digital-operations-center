```bash
# 调用M0进行需求分析

# 调用M4生成经营报表
/task M4 生成本月经营分析报告
```

**方式二: M0智能调度**
```bash
# M0会自动识别需求并调度对应智能体
/task M0 我需要分析上月经营数据并制定下月促销和采购计划
# M0会自动调度: M4(数据分析) → M2(促销方案) → M3(采购计划)
```

**方式三: MM战略规划**
```bash
# 复杂战略任务由MM统筹
/task MM 制定Q4季度美团管家中台建设规划
```

```
M0分析需求 → M1配置系统(餐厅/菜品/扫码点餐/外卖) → M2设置开业活动 → M3建立供应链 → M4配置报表
```

### 6.3 注意事项

**使用建议**:
1. 优先使用M0进行任务规划,避免直接调用专业智能体
2. 重大决策和跨组协调请升级给MM
3. 遇到模糊需求,先让M0澄清后再执行
4. 定期查看M4报表,数据驱动决策
5. 成本相关任务必须设定明确KPI目标

**常见错误**:
- ❌ 跨越M0直接调用M1-M4 (应先M0分析)
- ❌ 未明确验收标准 (应定义清晰KPI)
- ❌ 未考虑模块协同 (应考虑M1/M2/M3数据联动)
- ❌ 忽视成本控制 (应关注食材成本率≤30%)
